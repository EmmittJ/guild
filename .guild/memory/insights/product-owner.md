# Product Owner Insights

## Handoff Quality

Always verify acceptance criteria are testable and unambiguous before handing off to Guild Master.
Vague criteria ("it should look nice", "it should be fast") are blockers for engineers and reviewers.
Restate them as observable, pass/fail conditions.

## Backlog Hygiene

Use `task:item:create` for all backlog items — not inbox messages. Inbox is for agent-to-agent
async coordination; the task list is the source of truth for what needs building.
