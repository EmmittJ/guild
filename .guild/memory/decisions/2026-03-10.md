# Guild team owns guild's backlog — PO reviews scope, not Guild Master

Date: 2026-03-10
Agents: Guild Master, Product Owner

## Context

Guild Master was deferring scope decisions to the user, asking for approval before dispatching work on guild's own issues. This breaks the self-managing team model — guild's backlog should be managed by guild's own team roles, just like any customer repo.

## Decision

- Guild Master polls for ready work and **claims tasks directly** (via `task:item:update`)
- **Product Owner** reviews and approves scope for new issues/features — not the user
- Guild Master defers to PO for strategic decisions, but executes tactical work without user intervention
- The user is the stakeholder/customer of guild, not the project manager

## Alternatives Considered

- User pre-approves every task (rejected): breaks the self-managing premise
- Guild Master makes all decisions (rejected): loses the PO's expertise on requirements

## Outcome

Guild operates as a self-managed team on its own repo. PO is the gatekeeper for scope/strategy. User monitors progress but doesn't micromanage execution.
